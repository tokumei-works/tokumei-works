# Final Decision

## 結論

最初の収益化は **note有料ミニ商品販売** を主軸にします。

理由:

- 既存noteアカウントがある
- フォロワー数や再生時間などの収益化条件がない
- 匿名ペンネームで始めやすい
- 100円から販売でき、初収益までの距離が短い
- 商品、販売ページ、SNS告知、サイト導線を自動生成しやすい

GitHub Pagesは補助ではなく「母艦」として使います。noteだけだと分析や外部導線が弱いため、商品一覧、プロフィール、ポリシー、RSS、sitemap、SNS共有導線を自前サイトに置きます。

## 採用する構成

```text
data/products.json
        ↓
scripts/run_all.py
        ↓
dist/note_drafts/*.md
site/index.html + product pages + RSS + sitemap
dist/social_queue.csv
        ↓
note公開 / GitHub Pages公開 / SNS予約投稿
```

## 初回商品

1. 匿名ミニ収益化スターターキット
2. 30日分SNS投稿テンプレート
3. note有料記事テンプレート集

価格帯は最初 `500円から980円`。まず1件の販売を目的にし、売れたテーマだけ増補して単価を上げます。

## 後回しにするもの

- Google AdSense: 審査とコンテンツ量が必要。AI量産サイトは低価値判定のリスクが高い
- YouTube収益分配: 登録者・再生時間・Shorts再生数の条件が重い
- X収益分配: Premium、本人確認、インプレッション条件が重い
- SNSの完全自動投稿: API料金や規約変更が大きく、初期はキュー生成で十分

## 成功判定

最初の14日間は次を見ます。

- note有料記事のクリック数
- SNS投稿からのプロフィール遷移
- どの商品名がクリックされるか
- 無料部分の反応
- 1件でも購入が発生するか

1件売れたら、その商品を増補して第2版化します。売れなければタイトル、無料部分、価格を変えて再テストします。
